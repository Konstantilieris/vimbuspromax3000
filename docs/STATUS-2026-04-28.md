## TL;DR

- **M1 — "Execution Loop Live" — closed.** A failing evaluator verdict now drives retry → escalate → re-plan without operator nudging. Vimbus can execute against a generated task list end-to-end.
- **Planner quality** is feature-complete. Operators talk to the planner via a 5-round interview (scope → domain → interfaces → verification → policy) with persisted slices and live `planner.question` / `planner.answer` events.
- **Operator console** is ~80% done — SSE live stream, 3-pane TUI, operator notifications channel, dependency-map endpoint and CLI view, plus a Postgres `LISTEN`/`NOTIFY` adapter ready for two-process scale-out (built and tested, not yet wired into API startup).
- **Verification breadth** has not started. The browser-MCP wrapper and a11y/visual verification runtime (VIM-39, 8 pts) is the single largest remaining MVP gap.
- **Observability & production hardening** is 1/4 done — read-only DB-MCP wrapper landed; benchmark hydration, LangSmith export, and broader Postgres + prod hardening are still queued.

## Architecture at a glance

```
TaskGoblin / VimbusProMax3000 (Bun 1.3.13 monorepo)

apps/
  api/      HTTP + SSE — planner runs, executions, dependency-map, /events stream
  cli/      Live TUI — 4 panes (planner / executor / evaluator / operator notifications)

packages/
  agent/             Vercel AI SDK loop driver
  benchmarks/        Scenario + baseline scaffolding (queued for VIM-40)
  db/                Prisma client, repositories, LoopEventBus (in-process + Postgres adapter)
  evaluator/         runEvaluation; auto-flips ModelDecision.state on verdicts
  mcp-client/        MCP wrappers — taskgoblin-patch, taskgoblin-db (read-only), planner, ...
  model-registry/    ModelProvider / RegisteredModel / ProjectModelSlot
  observability/     Trace export surface (queued for VIM-41)
  planner/           5-round interview, multi-agent pipeline, project-manager pack
  policy-engine/     Policy evaluation
  shared/            LOOP_EVENT_TYPES + cross-package contracts
  task-intel/        Complexity scorer
  test-runner/       Red-green-refactor verification runner
  verification/      VerificationPlan / VerificationItem helpers
```

**Database**: PostgreSQL via Prisma. 29 models cover the operational core (`Project`, `PlannerRun`, `Task`, `TaskExecution`, `AgentStep`, `ModelDecision`, `LoopEvent`, `McpToolCall`) plus benchmark, regression, and observability scaffolding (`BenchmarkScenario`, `RegressionBaseline`, `LangSmithTraceLink`).

**Event flow**: every meaningful state change emits to `LoopEventBus`; the API forwards to SSE consumers, the TUI re-renders. Postgres `LISTEN`/`NOTIFY` adapter exists for cross-process delivery but isn't wired in yet (VIM-46).
