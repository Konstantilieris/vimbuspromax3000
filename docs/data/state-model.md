# State Model

Core loop: `Planner -> Verification Contract -> Approval -> Execution -> Verified Output`.

## Task Status

| Status | Meaning |
|---|---|
| `draft` | Generated but incomplete. |
| `planned` | Planner created a task proposal. |
| `awaiting_verification_approval` | Verification plan exists and needs operator approval. |
| `ready` | Task and verification plan are approved. |
| `executing` | Agent is working on the task branch. |
| `testing` | Verification is running. |
| `awaiting_patch_approval` | Verification passed and diff is waiting for review. |
| `verified` | Verification passed. |
| `completed` | Patch was approved and commit policy is satisfied. |
| `failed` | Task failed or retry budget was exhausted. |
| `abandoned` | Operator stopped the task. |

## Verification Plan Status

| Status | Meaning |
|---|---|
| `proposed` | Generated by planner or verification designer. |
| `approved` | Operator approved. |
| `rejected` | Operator rejected. |
| `superseded` | Replaced by a newer approved plan. |

## Execution Status

| Status | Meaning |
|---|---|
| `queued` | Ready to start. |
| `preparing_branch` | Branch creation or switch in progress. |
| `writing_tests` | Test-first materialization is running. |
| `confirming_red` | Runner is confirming red or pending state. |
| `implementing` | Executor is modifying code. |
| `verifying` | Runner is executing the contract. |
| `evaluating` | Evaluation engine is scoring verified output. |
| `retrying` | Policy selected another attempt. |
| `patch_ready` | Diff is ready for review. |
| `completed` | Patch approved and task complete. |
| `failed` | Execution failed. |

## Branch State

Branch states are defined in `architecture/branch-policy.md`.

## Approval Subjects

Approvals can target:

- planner run
- epic
- task
- verification plan
- verification item skip
- mutating tool call
- patch review
- branch abandon/reset
- model provider change

## Evaluation Status

| Status | Meaning |
|---|---|
| `queued` | Evaluation is waiting for verification results. |
| `running` | Evaluators are executing. |
| `passed` | Scores meet thresholds and no hard-fail dimension failed. |
| `warned` | Scores passed but warnings should be shown. |
| `failed` | Evaluation blocked patch review or retry. |

## MCP Tool Call Status

| Status | Meaning |
|---|---|
| `requested` | Agent requested the tool. |
| `approved` | Required approval was granted. |
| `blocked` | Policy blocked the tool call. |
| `running` | MCP server is executing the call. |
| `succeeded` | Tool call completed. |
| `failed` | Tool call returned an error. |

## Model Decision State

| State | Meaning |
|---|---|
| `selected` | Policy selected a model slot for the attempt. |
| `escalated` | Retry moved to a stronger slot. |
| `stopped` | Policy stopped retries. |

## Model Registry State

Provider status values:

| Status | Meaning |
|---|---|
| `pending_approval` | Provider config exists but cannot be used yet. |
| `active` | Provider can be used during slot resolution. |
| `disabled` | Provider is retained but excluded from resolution. |
| `error` | Provider validation failed and resolution should skip it. |

Secret references store labels and env variable names only. Operators must supply actual API keys through process environment.

## Regression Status

| Status | Meaning |
|---|---|
| `baseline` | Run is accepted as the comparison baseline. |
| `compared` | Run was compared against baseline. |
| `passed` | No blocking regression found. |
| `blocked` | Regression gate blocked the run. |
